## 功能特性

### 🎯 核心功能

| 功能模块 | 描述 |
|---------|------|
| **可视化编辑器** | PyQt5 拖拽式界面，无需编程即可创建自动化流程 |
| **OCR文字识别** | 支持中英文识别，智能点击指定文字位置 |
| **窗口/区域选择** | 一键选择目标窗口或自定义识别区域 |
| **循环执行** | 支持设置循环次数和间隔时间 |
| **内存优化** | 智能缓存管理，自动清理释放内存 |
| **全局热键** | F6 运行 / F9 停止（可自定义） |

### 🖱️ 鼠标操作

| 动作类型 | 功能描述 | 参数说明 |
|---------|---------|---------|
| **鼠标点击** | 在指定坐标执行点击操作 | `x`, `y`: 坐标位置<br>`button`: 左键/右键/中键<br>`clicks`: 点击次数 |
| **鼠标移动** | 移动鼠标到指定位置 | `x`, `y`: 目标坐标<br>`duration`: 移动耗时 |
| **鼠标拖拽** | 从起点拖拽到终点 | `start_x`, `start_y`: 起点<br>`end_x`, `end_y`: 终点 |
| **鼠标滚轮** | 模拟滚轮滚动 | `clicks`: 滚动格数<br>`direction`: 向上/向下 |

### ⌨️ 键盘操作

| 动作类型 | 功能描述 | 参数说明 |
|---------|---------|---------|
| **键盘输入** | 模拟键盘输入文本 | `text`: 输入内容<br>`enter_after`: 输入后按回车 |
| **按键操作** | 按下特定按键或组合键 | `key`: 单个按键<br>`keys`: 组合键如 ctrl+a |

### 🔍 图像识别

| 动作类型 | 功能描述 | 参数说明 |
|---------|---------|---------|
| **图像识别点击** | 通过模板匹配定位并点击 | `template`: 模板图片路径<br>`confidence`: 匹配置信度 |
| **查找图像** | 查找屏幕上的图像位置 | `find_all`: 查找所有匹配 |

### 📝 OCR 文字识别

| 参数 | 说明 |
|------|------|
| `click_text` | 点击包含此文字的区域 |
| `contains` | 检测是否包含此文字（不点击） |
| `exact_match` | 是否精确匹配 |
| `click_index` | 多个匹配时点击第几个 |
| `preprocess_mode` | 预处理模式：auto(快速)/all(准确) |

### 🔧 流程控制

| 动作类型 | 功能描述 |
|---------|---------|
| **延时等待** | 暂停执行指定时间 |
| **条件判断** | 根据条件跳转到不同步骤 |
| **循环控制** | 控制循环次数和跳转 |
| **执行脚本** | 执行 Python 代码片段（有安全限制） |

### 动作类型

#### 🖱️ 鼠标点击 (click)

**参数说明：**

|------|------|------|--------|------|

### OCR文字识别

#### 使用方法

1. **添加OCR步骤** - 从动作列表选择"OCR文字识别"
2. **设置识别区域** - 点击"📐 区域"按钮框选识别范围
3. **配置点击文字** - 在`click_text`填入要点击的文字
4. **选择匹配模式** - 勾选`exact_match`进行精确匹配

#### 参数详解

| 参数 | 说明 |
|------|------|
| `click_text` | 要点击的文字。OCR识别后点击包含此文字的区域中心 |
| `contains` | 只检测不点击，用于判断是否存在某文字 |
| `exact_match` | 勾选=完全一致，不勾选=包含即可 |
| `click_index` | 当有多个匹配时，点击第几个（0=第一个） |
| `preprocess_mode` | auto=快速(推荐)、all=全部模式(慢但准确) |
| `retry_count` | OCR重试次数，auto模式仅1次 |

#### click_text 与 contains 的区别

| 参数 | 作用 | 使用场景 |
|------|------|----------|
| `click_text` | 识别文字后**点击**该区域 | 点击按钮、链接 |
| `contains` | 只**检测**文字是否存在 | 条件判断、验证页面 |

#### 预处理模式说明

| 模式 | 速度 | 识别率 | 适用场景 |
|------|------|--------|----------|
| `auto` | ⚡⚡⚡⚡ 最快 | ⭐⭐⭐ 高 | 推荐，日常使用 |
| `all` | ⚡ 慢 | ⭐⭐⭐⭐ 最高 | 复杂图像，识别困难时 |
| `contrast` | ⚡⚡⚡⚡ 快 | ⭐⭐ 中 | 对比度低的图像 |
| `binary` | ⚡⚡⚡⚡ 快 | ⭐⭐ 中 | 背景复杂的图像 |

### 窗口与区域选择

#### 窗口选择

1. 点击工具栏 **"🪟 窗口"** 按钮
2. 选择 **"从列表选择窗口"**

3. 鼠标拖动选择区域
4. 自动应用到OCR步骤

**注意：** 窗口选择和区域选择互斥，选择一个会清除另一个

## 打包部署

### Windows 打包 EXE

```bash
# 安装打包工具
pip install pyinstaller

# 打包为单个EXE文件
pyinstaller --onefile --windowed --name "RabAI AutoClick" main.py

# 或使用配置文件
pyinstaller rabai_autoclick.spec
```

**打包参数说明：**

| 参数 | 说明 |
|------|------|
| `--onefile` | 打包为单个文件 |
| `--windowed` | 不显示控制台窗口 |
| `--name` | 输出文件名 |
| `--icon` | 设置图标 |

### macOS 打包 DMG

```bash
# 安装打包工具
pip install pyinstaller

# 打包为APP
pyinstaller --onefile --windowed --name "RabAI AutoClick" main.py

# 创建DMG（需要Mac环境）
hdiutil create -volname "RabAI AutoClick" -srcfolder dist/ -ov -format UDZO "RabAI AutoClick.dmg"
```

|------|----------|

## 常见问题

### Q: OCR 识别不准确？

A: 尝试以下方法：
- 缩小识别区域（使用"📐 区域"框选）
- 使用 `all` 预处理模式
- 确保文字清晰、对比度高
- 不勾选 `exact_match` 进行模糊匹配

### Q: 图像匹配失败？

A: 检查以下方面：
- 模板图片是否清晰
- 置信度是否合适（建议 0.7-0.9）
- 目标是否在屏幕可见区域

### Q: 快捷键不生效？

A: 确认：
- 已安装 `keyboard` 模块
- 没有其他软件占用相同快捷键
- 以管理员权限运行（Windows）

### Q: 如何在 macOS 上使用？

A: macOS 需要额外权限：
- 系统偏好设置 → 安全性与隐私 → 辅助功能
- 添加 Python 或终端到允许列表

### Q: 停止工作流不生效？

A: 点击停止后：
- 当前步骤会执行完成
- 循环会被中断
- UI状态会立即更新
